# Chapter Quality Checklist

**Purpose**: Manual quality gate for authored chapters before publication.
**Reference**: [spec.md](../specs/001-textbook-content-structure/spec.md)

## Pedagogical Structure
- [ ] 3-5 learning outcomes using Bloom's Taxonomy verbs (FR-006)
- [ ] Introduction provides context and "Why this matters" section
- [ ] Progressive complexity: logic flows from simple to complex concepts (FR-021)
- [ ] Hands-on exercise is practical and follows "Step-by-step" guidance (FR-009)
- [ ] 5-10 assessment questions included (FR-010)
- [ ] Estimated time included and realistic (FR-004)

## Technical Accuracy
- [ ] Minimum 2-5 code examples present (FR-007)
- [ ] Code examples are executable and have environment comments (FR-008)
- [ ] Inline comments explain non-obvious code logic
- [ ] Technical terms defined on first use or linked to Glossary (FR-018)
- [ ] Troubleshoot tips included for common exercise errors (FR-024)

## Visuals and Formatting
- [ ] Minimum 2 visuals (diagrams, screenshots) per chapter (FR-023)
- [ ] Visuals are numbered and captioned (FR-019)
- [ ] Correct Docusaurus/MDX syntax used for cross-references
- [ ] Header hierarchy is consistent (H1 for title, H2 for major sections)

## Final Review
- [ ] Chapter prerequisites are clearly listed (FR-017)
- [ ] "Next Chapter" link is correct and provides a preview
- [ ] Instructor guide (appendix) updated with answer keys
